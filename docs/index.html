GLYPH

Deterministic byte-exact retrieval over static corpora.

GLYPH is a retrieval primitive, not a search engine.

Why GLYPH exists

Most retrieval systems optimize interpretation.

GLYPH optimizes certainty.

Same bytes in → same matches out.

Core guarantees

  • byte-exact substring retrieval
  • deterministic results
  • no ranking
  • no fuzzy matching
  • no tokenization
  • no semantic interpretation
  • sentinel-safe FM-index construction

How it works

  1. Build suffix array + BWT + FM-index offline
  2. Load persistent index into memory
  3. Query exact bytes or hex patterns in ~ms

Use cases

  • large-scale log search
  • binary corpus lookup
  • forensics / DFIR
  • exact retrieval before vector search
