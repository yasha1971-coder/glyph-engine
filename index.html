GLYPH

Verifiable byte-exact retrieval over static corpora.

Current goal: first independent external verifier.

GLYPH is an experimental infrastructure project for deterministic exact byte matching. Its current focus is not ranking, search UX, or embeddings. Its current focus is reproducible verification.

Verify GLYPH in one command

git clone https://github.com/yasha1971-coder/glyph-engine.git
cd glyph-engine
./verify.sh

Expected:

VERIFY OK

This runs the canonical mini verification pipeline and checks that GLYPH builds a sentinel-safe FM-index over a fixed mini corpus and returns an exact match count of 2 for the query error.

If you run this independently, please report the result: contact@glyph.rs

What GLYPH is

GLYPH is a byte-exact substring retrieval engine over raw bytes.

No ranking. No embeddings. No fuzzy matching. No semantic interpretation.

Benchmark signal

HDFS 1GB, 100 exact blk_* lookups: 

grep repeated scan:        11.516 sec
GLYPH persistent FM:       0.001673 sec total
average:                   0.0167 ms/query

Experimental prototype. Not a grep replacement for one-off scans. Trades RAM and offline indexing for fast repeated exact queries over static corpora.
