Verifiable exact-byte retrieval over committed corpora.

GLYPH turns an exact binary query into deterministic coordinates, byte checks, portable evidence, and independent replay. It is designed for fixed corpora where reproducibility matters more than ranking, tokenization, or semantic similarity.

Evidence chain, not search results.

Search tools return matches. GLYPH records the corpus identity, exact query, deterministic coordinates, byte checks, and replay rules needed to independently reproduce the evidence result.

01 Committed corpus Corpus and source identities are fixed and recorded.

04 Byte checks Recorded coordinates are checked against source bytes.
05 Portable replay Evidence bundles can be independently inspected and replayed.

Where GLYPH sits.

GLYPH is not positioned as a faster grep, a SIEM, or a semantic retrieval platform. Its purpose is replayable exact-byte evidence over a committed corpus state.

System class Primary role Exact-byte retrieval Replayable evidence Relation to GLYPH
grep / ripgrep Fast local text search partial no GLYPH does not replace grep; it records a reproducible evidence result.
Elastic / Splunk Search, observability, and security analytics contextual no GLYPH is a narrow evidence layer, not an analytics platform.
YARA Textual and binary pattern matching yes no A possible upstream matcher for a future evidence workflow.
Sigstore / CT / OpenTimestamps Signing, transparency, provenance, or time no search different layer Potential complementary trust layers, not exact retrieval systems.
GLYPH verified path Replayable exact-byte retrieval evidence yes yes Verified semantics, compiled runtime, operator workflow, portable evidence, and replay.

Boundaries are part of the design.

GLYPH distinguishes exact retrieval evidence from source truth, legal conclusions, host security, and production-readiness claims.

Current strongest claim: these exact query bytes occurred at these exact canonical coordinates in this exact committed corpus state, under the verified retrieval and replay rules.

Not source truth or legal proof

Exact bytes do not establish authorship, intent, factual truth, admissibility, or complete chain of custody.

Not production or distributed infrastructure

The verified path is an engineering verification chain, not a production SLA or a petabyte-scale distributed deployment claim.

Not privileged-host protection

The current trusted-filesystem profile does not protect against a privileged writer mutating published inodes. Signing is specified but not implemented.

Independent reviewer path.

A reviewer can inspect the contract, run the complete verifier, challenge the committed result, and distinguish verified behavior from pending implementation.
